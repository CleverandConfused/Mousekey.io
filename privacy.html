Privacy Policy

Last updated: March 2026

Privacy Statement

Summary

MouseKey operates entirely offline and does not require an account.

Data collection

MouseKey does not collect, transmit, sell, or share personal data. Specifically no telemetry, no analytics, no advertising identifiers, no account sign-in, no background network communication.

Input handling

MouseKey interacts with Windows input only to provide its features. Mouse button events are detected to count click patterns and trigger assigned actions. Keyboard input is monitored to detect HID-mapped buttons (such as F13–F24) and is recorded only when you explicitly use the "Create Hot Key..." feature. MouseKey does not continuously log or monitor what you type. MouseKey records what the user types only after the user selects "Create Hot Key..." and the recording window is open. Recording is displayed when active. Once a key is pressed, recording starts. Recording stops automatically if another key is not pressed within 2 seconds. The user may restart recording, cancel, or save the recording for future activation.

Sensitive data notice

When creating custom hot keys or macros, any text the user records is stored locally and replayed when triggered. For security reasons, users should avoid recording passwords, one-time codes, private keys, or other sensitive information. MouseKey does not inspect, upload, or share the contents of recorded macros.

Data stored on your device

MouseKey stores configuration data locally under your Windows user profile. Stored data includes profile selections, enabled states, custom hot key definitions, custom mouse button mappings, and the turn-off button preference. This data is not intended to leave the device.

Network access

MouseKey does not require network access to function. If future versions introduce network-related features, this privacy statement will be updated accordingly.
